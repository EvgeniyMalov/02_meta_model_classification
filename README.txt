Построение модели кредитного риск-менеджмента (классификация)

Общее описание задачи:

Построить модель, предсказывающую факт допустит заемщик дефолт или нет 
(просрочка по кредиту более 90 дней).
Датасет представлен набором таблиц с крединой историей заемщика и 
файлом с целевой переменной.
Данные о кредитной истории представлены в закодированнном виде (категориальные несравнимые
переменные), у части признаков есть дубли - признаки-флаги.
Целевой показатель: ROC-AUC больше 0,75.


Общие этапы работы над проектом:

1.) Исходные данные обработаны (OHE) и сохранены (338 признаков). 0 класс - 2,9 млн, 1 класс - 0,1 млн
2.) Сокращение признаков до 222 (бинаризованные, те, у кого были аналоги с флагами);
3.) Сокращение признаков методом SVD (до 50 признаков), отдельно к нему задавал сбалансированные Class weights;
4.) Сокращение количества объектов Downsampling до 0,1 + 0,1 млн экземпляров обеих классов.
5.) Сокращение количества объектов Downsampling 0-го класса до 0,5 млн + upsampling 1-го класса до 0,5 млн.
6.) Сокращение количества объектов Downsampling 0-го класса до 0,3 млн + SMOTE 1-го класса до 0,3 млн.
7.) Обучение моделей с настройками по умолчанию и подбором гиперпараметров (lgbm, xgb, catboost),
применение мета-модели.
